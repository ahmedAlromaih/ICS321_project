# KFUPM Online Store

A Vite React website with a MySQL backend for the ICS321 KFUPM Online Store project.

## Overview

The system lets KFUPM users browse products, manage a cart, submit a payment form, and create orders in MySQL. Admin users can view store statistics, accounts, customer orders, and cancel orders.

## Features

- Sign in and sign up pages.
- Customer store with product search, cart, checkout, payment page, and order creation.
- Customer settings page with account details, account updating, order history, and a My Products seller page.
- Customer product selling with image URL, condition, price, quantity, category, and product editing.
- Seller order tracking with buyer contact links and delivery confirmation.
- Admin dashboard for summary statistics, all orders, account creation dates, and customer order history.
- Admin order cancellation with inventory restore.
- MySQL schema auto-checks for required product selling columns.

## Project Structure

```text

  db.js          MySQL connection pool and environment checks
```

## Requirements

- Node.js
- pnpm
- MySQL Community Server
- A MySQL database named `kfupm_online_store`

## Environment

Create `.env` in the project root. Use `.env.example` as the template.

```env
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=your_mysql_password
DB_NAME=kfupm_online_store
DB_PORT=3306
DB_SSL=false
```

Do not commit `.env` because it contains your database password.

## Database Setup

In MySQL CMD:

```sql
CREATE DATABASE IF NOT EXISTS kfupm_online_store;
USE kfupm_online_store;
SOURCE C:/Users/ahmed/Desktop/ICS321_project/api/schema.sql;
```

The expected tables are:

```text

cart_item

orders

```

## Run The Website

Install dependencies if needed:

```bash
pnpm install
```

Build and start the local server:

```bash
pnpm build
pnpm start
```

Open:

```text
http://localhost:3001
```

For development, run the API and Vite in two terminals:

```bash
pnpm api
pnpm dev
```

## API Documentation

The local server exposes these routes:

```text
GET    /api
GET    /api/summary
POST   /api/auth
POST   /api/signup
GET    /api/products
POST   /api/products
PATCH  /api/products
GET    /api/categories
GET    /api/customers
PATCH  /api/customers
GET    /api/cart
POST   /api/cart
PATCH  /api/cart
DELETE /api/cart
GET    /api/orders
POST   /api/orders
PATCH  /api/orders
GET    /api/reviews
POST   /api/reviews
```

Seller order tracking uses:

```text
GET /api/orders?seller_customer_id=1
```

Confirm delivery uses `PATCH /api/orders` with:

```json
{
  "action": "confirm_delivery",
  "order_id": 1,
  "customer_id": 1
}
```

## Product Images

Product images should use a real browser-loadable image URL. For local images, put the file in `public/images` and store a path like:

```text
/images/calculator.jpg
```

Remote image URLs should point directly to an image file such as `.jpg`, `.png`, or `.webp`.
